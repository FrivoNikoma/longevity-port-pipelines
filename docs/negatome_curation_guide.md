# NEGATOME-style negative-control curation guide

This document defines practical curation rules for preparing NEGATOME-style negative-control partner inputs for the SIRT6 mini-pilot.

It complements:

- `docs/negatome_control_inputs.md`
- `scripts.validate_negatome_control_inputs`
- `scripts.generate_negatome_control_pair_candidates`

The goal is to avoid treating arbitrary random proteins as valid negative controls.

## 1. Purpose

The mini-pilot currently reports:

- shuffled mask control ratios;
- NEGATOME-style control ratio fields;
- negative-control audit status;
- control-aware candidate scorecards;
- NEGATOME-style candidate scaffold rows.

However, true NEGATOME-style controls are not yet populated.

The current correct interpretation remains:

```text
shuffled mask control: implemented
NEGATOME-style control: not yet populated
candidate scaffold: generated for curation only
scorecard status: missing_negatome

```

This guide describes how future curated rows should be selected before they are allowed into:

```text
data/interim/negatome_control_pairs.csv

```

## 2. What counts as a NEGATOME-style negative partner?

A valid negative-control partner should be a protein sequence that is expected not to form the tested interaction with the source protein under the relevant biological or structural context.

A good negative-control partner should ideally satisfy several conditions:

- it is biologically plausible enough to be a meaningful control;
- it is not the known binding partner in the selected complex;
- it is not a close paralog of the true binding partner;
- it is not part of the same well-established complex for this interaction context;
- it has no known or obvious direct interaction with the source protein in the tested context;
- it has a sequence available for embedding;
- its choice can be explained in a short curation note.

The control should be negative for the interaction being tested, not necessarily negative for all possible interactions in biology.

## 3. What should not be used?

Do not use a protein as a negative partner if it is:

- the actual partner chain from the source complex;
- a species ortholog of the actual partner;
- a close paralog or homolog of the actual partner;
- a known member of the same complex;
- known to bind the source protein in the same biological context;
- a fragment, synthetic construct, or ambiguous sequence with unclear provenance;
- a sequence with missing or low-confidence identity;
- a random protein selected only because it is available.

A random protein is usually a weak control. It may be useful for a background stress test, but it should not be labeled as a curated NEGATOME-style control unless the rationale is explicit.

## 4. Required input columns

Curated rows must satisfy the input contract from `docs/negatome_control_inputs.md`.

Required columns:

```text
complex_id
chain
target_species
source_uniprot
negative_partner_uniprot
negative_partner_source
negative_partner_sequence
control_type

```

The candidate scaffold generated by:

```powershell
uv run python -m scripts.generate_negatome_control_pair_candidates

```

contains additional curation helper columns, but the final input file must provide the required contract columns.

Expected final input path:

```text
data/interim/negatome_control_pairs.csv

```

## 5. Recommended curation workflow

Start from the generated scaffold:

```text
data/output/sirt6_mini_pilot_negatome_control_pair_candidates.csv

```

This scaffold is generated from residue-delta outputs and contains one candidate row per mini-pilot complex, chain, and target species.

The scaffold rows are intentionally marked as:

```text
ready_for_input_contract = false
negative_partner_source = curation_required

```

To curate a row:

1. choose a negative partner candidate;
2. verify that it is not the true partner or close homolog of the true partner;
3. verify that it is not a known interacting partner in the same context;
4. fill `negative_partner_uniprot`;
5. fill `negative_partner_sequence`;
6. set `negative_partner_source`;
7. set `control_type`;
8. optionally keep an external curation note outside the strict input contract;
9. run the input validator.

Validation command:

```powershell
uv run python -m scripts.validate_negatome_control_inputs

```

## 6. Suggested control_type values

Use controlled vocabulary where possible.

Recommended values:

```text
curated_non_interactor
database_supported_non_interactor
literature_supported_non_interactor
orthogonal_pathway_control
background_protein_control

```

Use `background_protein_control` only when the protein is a deliberately weak/random background control and not a strong NEGATOME-style non-interactor.

Do not use:

```text
negatome_style_curation_required

```

in the final `data/interim/negatome_control_pairs.csv`. That value belongs to the candidate scaffold only.

## 7. Suggested negative_partner_source values

Recommended values:

```text
manual_curation
negatome_database
literature_curation
intact_or_biogrid_absence_screen
orthogonal_pathway_selection

```

If the partner was selected only as a placeholder, do not place it into the final input file.

## 8. First mini-pilot targets for curation

For the first real curation pass, prefer a small subset rather than all 15 rows.

Good first targets:

```text
SIRT6 receptor / mouse
Ku70 receptor / mouse
PARP1 ligand or receptor / mouse

```

Rationale:

- mouse is experimentally interpretable;
- these candidates are central to the mini-pilot;
- they allow testing the input contract with a small number of curated rows;
- they reduce the risk of silently propagating low-quality controls across all outputs.

Do not curate all species until the first small subset passes validation and downstream interpretation remains stable.

## 10. How to interpret partial curation

If only some rows have curated controls, the audit should distinguish:

```text
has_shuffled_and_negatome
missing_negatome
missing_shuffled
missing_all_controls

```

Partially populated NEGATOME controls should not be interpreted as global validation of the whole mini-pilot.

The scorecard should remain control-aware at row level.

## 11. Relationship to shuffled controls

Shuffled mask controls and NEGATOME-style controls answer different questions.

The shuffled mask control asks whether the interface mask itself produces a stronger signal than a shuffled residue mask.

The NEGATOME-style control asks whether the candidate interaction signal remains meaningful against a biologically curated non-partner or negative partner.

Both are useful, but neither replaces the other.

A candidate with only shuffled support should remain preliminary.

## 12. Do not commit generated outputs by default

Files under:

```text
data/output/

```

are generated artifacts and should generally not be committed.

The scaffold:

```text
data/output/sirt6_mini_pilot_negatome_control_pair_candidates.csv

```

is useful for local curation but should not normally be committed.

The curated input:

```text
data/interim/negatome_control_pairs.csv

```

may be committed only if the project decides that curated negative-control inputs belong in the repository.

Before committing any curated input, review whether it contains:

- only public identifiers and sequences;
- no private notes;
- no unreviewed speculative labels;
- no generated placeholder rows.

## 13. Minimum acceptance criteria for a curated input row

A row can be considered ready for the input contract only if:

- `complex_id` matches a mini-pilot result row;
- `chain` matches the source chain role;
- `target_species` matches a mini-pilot target species;
- `source_uniprot` matches the tested source protein;
- `negative_partner_uniprot` is non-empty;
- `negative_partner_sequence` is non-empty;
- `negative_partner_source` is non-empty and not `curation_required`;
- `control_type` is non-empty and not `negatome_style_curation_required`;
- the selected negative partner has a clear curation rationale.

Rows that fail these criteria should stay in the scaffold or external notes, not in `data/interim/negatome_control_pairs.csv`.

## 14. Recommended first-pass procedure

For the first curated mini-pilot test:

1. select 2-3 mouse rows from the scaffold;
2. curate negative partners manually;
3. create a small `data/interim/negatome_control_pairs.csv`;
4. run:

```powershell
uv run python -m scripts.validate_negatome_control_inputs

```

1. confirm that only the curated rows are marked as having negative-control input;
2. do not yet change enrichment ratios;
3. only after validation, implement the step that embeds negative partners and computes `negatome_control_ratio`.

## 15. Important interpretation note

A valid input table does not automatically mean that NEGATOME-style ratios are populated.

The pipeline has three distinct states:

```text
candidate scaffold exists
curated input table exists
computed NEGATOME-style control ratios exist

```

Only the third state should change enrichment-level `negatome_control_ratio` values.

Until then, downstream scorecards should continue to treat candidates as missing full NEGATOME-style control evidence.
